What you need (all free)
1

A Base wallet required

Download MetaMask from metamask.io or Coinbase Wallet. Switch the network to "Base" inside the app. Your wallet address starts with 0x and is safe to share publicly.

2

Basescan API key free

Go to basescan.org/register and create a free account. Then go to your account settings and click "API Keys" to generate one. This lets the app read your public transaction history.

3

Claude API key free credits

Go to console.anthropic.com, sign up, and go to "API Keys" to create one. New accounts get free credits. This is what generates the actual tweet text.

4

That's it — no installation needed

This is a pure HTML file. Just open it in any browser. No servers, no frameworks, no command line.

How the app works
1

You enter your wallet address

The app takes your public 0x address — no password or private key needed.

2

Fetches your transactions from Basescan

Using the Basescan API, the app retrieves your recent transaction history — counts, ETH volume, and contract interactions.

GET https://api.basescan.org/api?module=account&action=txlist&address=0x...&apikey=YOUR_KEY
3

Sends the data to Claude

The transaction summary is sent to the Claude API with instructions to write a tweet in your chosen tone.

4

Displays the tweet

You get a ready-to-post tweet with a copy button and direct link to post it on X.

This app runs entirely in your browser. No data is stored on any server. Your API keys stay in your browser's memory and are cleared when you close the tab.
What is a GitHub commit?

A commit is a saved change to your code. Every time you update a file on GitHub and save it, that's one commit. Building and improving this project over time is the most natural way to build up your commit count on Base.

Step 1 — Upload this file to GitHub
1

Create a GitHub account

Go to github.com and sign up for free.

2

Create a new repository

Click the green "New" button. Name it something like base-tweet-generator. Set it to Public so others can see it. Click "Create repository".

3

Upload index.html

Click "Add file" → "Upload files". Drag and drop the index.html file you downloaded. Add a commit message like "Initial upload" and click "Commit changes". That's your first commit.

Step 2 — Keep making commits by improving the app

Every improvement you make and save to GitHub = one more commit. Here are easy things to improve:

→

Change the color scheme

Edit the CSS variables at the top of the file. Change --accent: #0052ff to a different color. Save and commit.

→

Add a new tweet tone

In the HTML, find the <select id="tone"> section and add a new <option>. Then update the toneMap in the JavaScript.

→

Add a README

Create a new file called README.md in your repo and describe what the project does. That's another commit.

→

Fix bugs or add features

Use Claude to help you add new features — like showing NFT activity, or translating the tweet to another language.

Step 3 — Host it live (optional but impressive)
1

Go to Settings in your repo

Click "Settings" at the top of your GitHub repository.

2

Enable GitHub Pages

Scroll down to "Pages", set source to "Deploy from a branch", select "main" branch, and save. Your app will be live at yourusername.github.io/base-tweet-generator.

Remember: real commits come from real improvements. Making small, genuine changes over time is how developers actually work — and it's exactly what gives you a healthy commit history.
